# AI Software Factory - Restructuring Notes

## What Changed

The AI Software Factory has been restructured to support **dynamic schema generation** and **independent phase execution**.

## Key Changes

### 1. **Database Schema is Now Dynamic**

**Before:**
- Pre-defined 13-table schema
- Same schema for all projects
- Not specific to requirements

**After:**
- Minimal factory database (just project metadata)
- Each project generates its own schema during Database Design phase
- Schema stored as JSON in project metadata
- Schema is versioned and can be regenerated

### 2. **Removed Pre-defined Tables**

The following tables are no longer pre-defined:
- `requirements` - Requirements are domain-specific
- `tasks` - Tasks are generated based on architecture
- `code_files` - Generated by code generation phase
- `test_cases` - Generated by test generation phase
- `bugs` - Generated by bug detection phase
- `reviews` - Generated by code review phase

**Kept for Factory Operation:**
- `project_metadata` - Track projects
- `artifacts` - Store all generated artifacts
- `execution_history` - Track phase executions

### 3. **Pipeline Executor Added**

New `pipeline/executor.py` module provides:
- 15 SDLC phases defined with dependencies
- Independent phase execution
- Dependency checking
- Phase ordering
- Individual phase status tracking

### 4. **Schema Generator Added**

New `database/schema_generator.py` module provides:
- Dynamic schema generation
- Schema as data structure
- SQL script generation
- Schema versioning
- Example schemas (todo-app, ecommerce)

### 5. **CLI Enhanced**

New commands:
```bash
python main.py list-phases              # List all available phases
python main.py run-phase ...            # Run single phase
python main.py run-pipeline ...         # Run multiple/all phases
```

Enhanced commands:
```bash
python main.py run-pipeline --phases "phase1,phase2"  # Run specific phases
python main.py run-phase --force                       # Skip dependency check
```

## How It Works Now

### Project Creation

```bash
python main.py create-project \
  --name "todo-app" \
  --requirement "Build a task management app"
```

Creates:
- Project metadata entry
- Project directory
- Ready for phase execution

### Phase Execution

```bash
# View available phases
python main.py list-phases

# Run requirements analysis
python main.py run-phase --project "todo-app" --phase requirements_analysis

# Run database design (generates schema)
python main.py run-phase --project "todo-app" --phase database_design

# Run full pipeline
python main.py run-pipeline --project "todo-app"
```

### Schema Generation Workflow

1. **Requirements Analysis Phase**
   - Analyzes business requirement
   - Extracts entities and relationships
   - Produces domain model

2. **Domain Modeling Phase**
   - Creates detailed domain model
   - Defines constraints and rules
   - Maps business concepts to entities

3. **Database Design Phase**
   - Reads domain model
   - Generates optimal schema
   - Creates relationships and indexes
   - Produces SQL script
   - Stores schema in project metadata

4. **Subsequent Phases**
   - Use generated schema
   - Design APIs based on entities
   - Generate code for data access
   - Create tests for database

## Benefits of This Approach

### 1. **No Pre-defined Constraints**
- Each project gets exactly what it needs
- No unnecessary tables or columns
- Optimized schema per requirements

### 2. **Auditable**
- Schema is stored and versioned
- Can see how it was derived
- Full lineage from requirements to schema

### 3. **Flexible Execution**
- Run any phase independently
- Skip unnecessary phases
- Re-run phases if needed
- Run in custom order (with --force)

### 4. **Scalable**
- Handle projects of any complexity
- Thousands of tables if needed
- Custom constraints and rules
- Domain-specific optimizations

### 5. **Traceable**
- Every artifact is version controlled
- Know which phase produced what
- Full execution history
- Can replay pipeline

## Migration Path

### For Existing Projects

If you have existing projects, you'll need to:

1. Create new project with same requirement
2. Run requirements and database design phases
3. Schema will be generated automatically
4. Can delete old project if needed

### For New Projects

Just use the new workflow:
```bash
python main.py create-project ...
python main.py run-pipeline ...
```

## FAQ

### Q: Where is the database schema defined?
A: It's generated during the Database Design phase and stored in `project_metadata.generated_schema` as JSON.

### Q: Can I modify the generated schema?
A: Yes, the schema is stored as JSON. You can edit it manually before code generation.

### Q: Can I run phases out of order?
A: Use `--force` flag to skip dependency checks. Example: `--phase api_design --force`

### Q: What if I want to regenerate the schema?
A: Run the database design phase again with `--force` to overwrite the existing schema.

### Q: How do I know what each phase does?
A: Run `python main.py list-phases` to see all phases with descriptions and dependencies.

### Q: Can I run just one phase?
A: Yes, use `python main.py run-phase --phase <phase-id>`

### Q: What happens if a phase fails?
A: The pipeline stops. Re-run the phase or use `--force` to retry.

### Q: Can I skip some phases?
A: Yes, use `--phases "phase1,phase3"` to run only those phases in the pipeline.

## Backward Compatibility

This restructuring is **not backward compatible** with previous versions. You'll need to:

1. Delete old database (`factory.db`)
2. Re-create projects using new workflow
3. Update any custom agents to use new pipeline executor

The new structure is much more flexible and scalable, so the migration is worth it!
